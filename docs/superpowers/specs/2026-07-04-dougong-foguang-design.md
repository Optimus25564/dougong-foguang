# 《斗拱：重建佛光寺》设计文档

- 日期：2026-07-04
- 状态：设计已与用户对齐，待用户复核
- 作者：Nana + Claude

## 一句话概念

一个网页 3D 游戏。玩家从零件盘中把**佛光寺东大殿柱头铺作（七铺作·双杪双下昂）**逐层拼起；每放对一件，镜头拉近、部件高亮、剖面与受力动画讲清它是什么、干什么、怎么和邻件咬合。拼完一整朵斗拱，玩家便"从手上"理解了唐构中屋檐之力如何一层层传到柱子。

## 目标受众

建筑爱好者 / 用户本人。术语精确到位（铺作、卷杀、偷心/计心、斗口为模等），不回避专业性。中文界面。

## 已确认的设计决策

| 维度 | 决策 |
|------|------|
| 范围 | 聚焦**单朵斗拱**：东大殿柱头铺作（七铺作双杪双下昂）。不做整栋建筑。 |
| 玩法 | **引导式逐层拼装**为主（高亮下一件虚影→拖拽对位→卡扣归位）；后期加**挑战模式**（去提示、放错失稳）作为"一点 B"。 |
| 教学 | **3D 镜头拉近 + 部件高亮 + 剖面/受力动画 + 精确旁注**（沉浸式，方案 B）。 |
| 美术 | **写实唐代木构**：几何按实测比例，PBR 木料 + 程序化木纹，环境光遮蔽 + 柔和光照。"要做就做到最好"。 |
| 几何来源 | **参数化程序生成**：实测尺寸→数据表→代码按营造法式比例逐件生成（含卷杀曲线、斗欹）。不依赖外部建模。 |
| 技术栈 | Three.js + Vite + Vitest，模块化。沿用用户火箭游戏项目的结构约定。 |

## 核心循环（单个部件）

```
看提示(下一件高亮虚影) → 从零件盘拖起 → 旋转对位 → 卡扣归位(咔哒+木声)
        ↓
镜头拉近 + 部件高光 → 精确旁注讲解卡 → 受力/咬合动画演示
        ↓
图鉴解锁该件 → 高亮下一件虚影 → 循环
```

拼完全部 → **整朵斗拱受力总览动画**（力从撩檐槫经昂、拱、斗，汇到栌斗、落柱）→ 解锁**挑战模式**。

## 内容与数据模型

### 部件清单（自下而上，主轴）

以梁思成《记五台山佛光寺的建筑》实测数据与《营造法式》为准；实现前逐条核对来源，此处为结构框架，精确的计心/偷心分布、斗数、尺寸在实现时以权威资料定稿。

1. **栌斗**（lúdǒu）— 柱头之上的大坐斗，全朵之基，承всё上部之力。
2. **华拱·第一杪**（huágǒng）— 出跳横栱，佛光寺此跳**偷心**。
3. **华拱·第二杪** — 第二出跳，跳头置交互斗。
4. **下昂·第一昂**（xiàáng）— 下斜悬挑的杠杆构件。
5. **下昂·第二昂** — 第二重下昂。
6. **令拱**（línggǒng）— 最上横栱，承替木/撩檐槫。
7. **昂形耍头**（shuǎtóu）— 佛光寺耍头作昂形。
8. **衬方头**（chènfāngtóu）。
9. 配套小件：**交互斗、散斗、齐心斗**（各跳头）、**瓜子拱/慢拱**（计心跳）、**替木 + 撩檐槫**（顶）。

}
```

### 教学高潮：下昂杠杆原理

下昂是全场教学重点：挑檐重量经昂身、以栌斗一线为支点，被上部屋架压住的昂尾所平衡——杠杆。此受力动画重点打磨。

## 架构（模块）

沿用火箭游戏结构约定：

```
src/
  main.js                 入口，装配各模块
  content/
    parts.js              部件数据表（唯一真相来源）
    assemblySteps.js      拼装顺序、每步提示与校验
  scene/
    dougongScene.js       Three.js 场景、相机、灯光、地面
    geometry/             各部件参数化几何生成器
      ludou.js  gong.js  ang.js  dou.js  ...
    materials/woodPBR.js  程序化木纹 PBR 材质
  interaction/
    dragPlace.js          从零件盘拖拽、旋转、吸附归位
    snapValidator.js      对位与咬合校验（含挑战模式失稳判定）
  teach/
    cameraFocus.js        镜头拉近/环绕
    highlight.js          部件高光/虚影
    forceAnim.js          受力/咬合动画（下昂杠杆等）
    annotation.js         旁注讲解卡
  ui/
    hud.js  tray.js  codex.js（图鉴）  styles.css
  audio/                  卡扣木声等
tests/                    vitest：几何尺寸、拼装顺序校验、吸附逻辑
```

模块边界原则：`content/` 是唯一数据真相，`scene/geometry/` 只读数据出几何，`interaction/` 只管输入与吸附，`teach/` 只管演示，互不越界，可独立测试。

## MVP 范围

**做进 MVP：**
- 主轴拼装跑通：栌斗 → 双杪华拱 → 双下昂 → 令拱（含各自吸附归位）
- B 式教学框架：镜头拉近 + 高亮 + 旁注；**关键三件（栌斗、华拱、下昂）有受力动画**，其余先高亮+文字
- 图鉴：拼一件解锁一页，可回看
- 一个整朵**受力总览动画**
- 写实木料 PBR 材质与光照

**MVP 之后：**
- 散斗/交互斗/齐心斗、瓜子拱/慢拱、替木撩檐槫等配套小件
- 全部部件的受力动画
- 挑战模式（打乱、去提示、失稳）
- 音效打磨、参数可调（"把斗口调大 0.5 寸看变化"）

## 测试策略

Vitest 单元测试：
- 几何生成器输出的关键尺寸符合数据表（如栌斗长宽高、拱的卷杀瓣数）
- 拼装顺序与父件依赖自洽（不会要求先放上层）
- 吸附校验：正确位置吸附、错误位置拒绝
- 数据表完整性（每件必填字段齐全）

场景渲染与动画为手动/视觉验证。

- 不做外部 Blender 建模管线
